Moderation instructions

  • Account

    Global moderation

    Any account has a numeric global role assigned to it. Any role under 4 is global.

    • Root: 0
    • Admin: 1
    • Volunteer: 2
    • Janitor: 3

    Anything above 3, or undefined, is considered a regular user. Janitors are able to delete any post or thread on the site and to close global reports.
    Volunteers are able able to lift and issue any kind of global ban.

    Roots are able to assign admins.
    Admins are able to add or remove volunteers and janitors and to delete and transfer boards. They are also able to create boards even when board creation is disabled.
    The first admin or root will have to have its role set using the terminal arguments. Consult the readme.md in LynxChan's src/be for instructions on that. Any role has all the powers of the less privileged] roles. A link to the global staff tools is found on the Account page.

    Board moderation

    A board has a board owner and board volunteers. Board volunteers are added by the board owner and are able to delete content on the board, lock, pin and make threads cyclic, close board reports and apply bans for the board.
    Board owners can change the board settings and add rules, filters, banners, and custom css. They can also transfer the board ownership or delete the board.

    Custom CSS

    Board owners can add a css file to their boards by using the upload form on the board management page. It will be applied after the standard CSS and thus override its declarations (keep in mind the CSS specificity rules though).
    To delete the custom css file, click the upload button without entering any files.

    Reports

    Any non-banned user is able to report content.
    Open reports will be displayed on the main moderation page with a link to the reported content and an option to close the report.
    Closed reports are archived on the closed reports page.

    Deleting content

    As long as you are logged in, your global role and board role will be automatically verified when you try to delete content AS LONG AS YOU DON'T INPUT A PASSWORD.
    If you input a password, your account will not be validated and the system will try to use the regular deletion password you provided.

    Acessing the moderation tools

    On each thread is a link that will lead you to the moderation page for that board and thread.

    Thread settings

    Threads have 3 moderator settings: pin, lock and cyclic.
    Pin will make a thread be listed above all others, no matter how long it was it's last bump.
    Lock will prevent anyone from posting on a thread.
    Cyclic will make the thread never go into auto-sage mode. Instead it will delete old posts when it hits the post count limit that would trigger auto-sage.

    Bans

    To apply bans, select the checkboxes on the left top corner of postings and write an expiration for the ban. The date format must be parseable using the date parse of javascript.

    Examples

    01/02/2016 will ban the user until February 1st of 2016.
    01/02/2016 12:30 will ban the user until the same date at 12:30 of the same day.
    Bans can be lifted from the bans page that can be accessed from the moderation pages.
    Optionally, you can enter a ban reason and message. The ban message will be displayed on the post itself instead of the default ban message set in the site configurations. Global staff with the power to apply bans can select the checkbox to apply a global ban.
    The ban reason will appear to the user and on the bans list.

    IP range bans

    It is also possible to ban entire IP ranges based on its first two parts. Notice that the range of the poster's IP can be seen on the moderation page for threads. Enter this range on the box in the range bans page and select to apply it.

    File bans

    Files can also be banned, keeping anyone from posting them. Copy the md5 hash of the file shown in the thread moderation page and input it on the hash bans page. Keep in mind that the slightest modification to a file will change its hash.

    Board settings

    There are many ways to customize how a board works in general.
    The board name, its description, and the default name for anonymous users can be changed. Disable captcha will disable captcha for the board completely. Keep in mind that this will expose the board to flood since there are no anti-flood tools in place yet.
    Force anonymity will disable the name field completely. Role signature can still be used by appropriate users.
    By default users can be identified inside threads by an unique ID that will use the user's IP. This behavior can be disabled by checking the option to disable thread-wise IDs.

    Filters

    Filters automatically swap a term for another term on newly posted posts. This isf useful to censor potentially triggering content.

    Banners

    The board owner can add customized banners to their board, they will be displayed randomly for users.

    Rules

    Rules can be written and will be made visible on the board's rules page.
